在某些场景下你可能需要在页面中直接输出Excel中的数据，或者是某些csv格式文件中的数据。Freezes 提供了一个简单的数据入口，可以将外部数据文件读入至页面的上下文对象内，并通过 site.data.文件名 获取。

只要将数据文件放置于Freezes 站点根目录下的data内, Freezes 在运行时将会自动把数据文件加载至 site.data 变量内，提供模板内引用。 Freezes 可以支持 YAML, JSON 和 CSV 格式的数据格式文件。(注：CSV文件的首行必须为列名称)

为了更好地解释如何来使用这个强大的文本化数据特色，以下将分别以 YML, JSON 和 CSV 数据文件的使用举例：

示例: 成员列表

首先分别准备YML或CSV格式的数据文件，本例名为 members.yml/members.csv 并放置于 data 目录中

data/members.yml:

- name: Ray
  github: dotnetage

- name: Edison
  github: edison

- name: Money
  github: money

或者使用 data/members.csv:

name,github
Ray,dotnetage
Edison,edison
Money,money

接下来就可以在页面布局模板中通过 site.data.members 访问文件内的数据了。

注: 变量对象的名称就是文件名

<ul>
{% for member in site.data.members %}
  <li>
    <a href="https://github.com/{{ member.github }}">
      {{ member.name }}
    </a>
  </li>
{% endfor %}
</ul>

示例: 组织

数据目录中可以使用子目录，所有的子目录会被 Freezes 作为一个对象加载至 site.data 变量之内，如果你的数据文件很多的时候这会是一个非常有用的功能，被创建的目录对象可以当作分类对象或者命名空间使用。

数据文件 data/orgs/freezes.yml:

username: freezes
name: freezes
members:
  - name: Ray
    github: dotnetage

  - name: Edison
    github: edison

再增加一个数据文件 data/orgs/doeorg.yml:

username: doeorg
name: Doe Org
members:
  - name: John Doe
    github: jdoe

此时 orgs 这个目录就可以通过 site.data.orgs 来访问:

<ul>
{% for org_hash in site.data.orgs %}
{% assign org = org_hash[1] %}
  <li>
    <a href="https://github.com/{{ org.username }}">
      {{ org.name }}
    </a>
    ({{ org.members | size }} members)
  </li>
{% endfor %}
</ul>
