我们的设计哲学认为：“文档即是数据”所以 Freezes 是一个无数据库的环境。markdown 文稿就是Freezes的数据。您可以通过Markdown Extra 扩展规范中的 元数据（metadata）告诉 Freezes 应该以何种方式显示的文稿，诸如 文章标题 , 封面 ， 页面布局 ，主题词 。

什么是元数据

元数据属于 Markdown Extra 的一种可用于seo的特殊格式文法，元数据的内容一般不会被显示于正文中，对于控个用于控制页面输出的特定元数据外，其它的元数据可以通过页面变量访问。

元数据的定义需要在文本的首行加入，与正文之间通过一个空行分隔，示例如下：

title: 标题
description: 文章的简述
author: 作者

此处为文章正文

Freezes 所支持的元数据如下：

元数据 说明
title 文章与页的标题（必选），如果不填将不会显示于菜单内。
description 输出至页面 <meta> 标记内的文字，限定为 255个字符
summary 文章的摘要，显示于文章列表中，也可以作为副标题使用
cover 文章封面图片地址，此处不需要声明文件的全路径，只需将图片放置于与文本文件同名目录下即可，具体请见下文[链接资源]
author 文章的作者名称
date 文章的发表日期，必须以 yyyy-MM-dd hh:mm:dd 的严格格式设定，如 2015-01-01 00:00:00。如果不填写此日期 Freezes 将以文本的最后更新日期作为发表日期。
layout 指定文本以何种布局页显示，该名称指向与templates/layouts目录下同名的页面，指定时无须后缀名。不指定时 Freezes 将智能识别应采用何用布局。
tags 文章的主题标签，多个标签可通过 , 分隔，指定后可在 /tags 页面内显示并被过滤筛选。添加后会自动输出至页面的 <meta name="keyword" /> 元数据标签内增强SEO效果。
id 指定文章的索引号，在 posts 目录外的所有的文章会按此值由小到大排序
注意
  • 元数据标签: 后面必须紧跟一个空格，否则将被视为格式错误。
  • 如果元数据内需要采用由字母与数字之外的符号（如中文）请包括在 “” 之内

使用建议

  • 每个文件都增加 tags 与 description 或 summary 这两元数据将自动输出至页面的 <meta> 内，以增加页面的SEO效果。
  • 为每个文件增加发布日期，可以方便于文章的排序
  • tags 应该准确，且应由大分类向小分类的方式排序，且不适于标记超过10个以上的关键词，以避免降低SEO的效果。
