全球化内容

一个站点，多个语言版本

Freezes 的与别不同之处是他可以生成多语言的静态化站点。那么对于有全球化需要的站点，这将会是一个很棒的功能！正如我们的官网一样。

全球化内容并不要求做什么设置或者学习一些什么额外的技能。你只需要采用特殊的目录命名就可以完成这一切！

唯一需要了解的只有两点：

  • 其它语言版本的首页
  • 区域化目录

Freezes 将 pages 目录下的所有的文件视为使用 默认语言，遇到以区域化规则命名的目录则视为使用区域性语言。

例如，当前的站点是中文的，而需要增加一个英语版本的站点，则只需要在pages下增加一个名为en-us区域化目录，而需要在 pages 目录内增加一个名为 en-us.md 的文件作为该语言版本的 默认页。那么当用户访问 这个中文站时 Freezes 将不会加载 en-us.md 文件到主菜单内，而需要访问英文站点只需要直接访问 http://localhost:5500/en-us/ 将自动转入至英文站。

文件结构如下所示：

.
├── pages // 默认站点
|   ├── en-us // 英文站点
|   |     ├── my-design-philosophy.md
|   |     ├── ...
|   |     └── why-programmer-should-write-books.md
|   ├── de-de // 德文站点  
|   ├── en-us.md  // 英文站首页  
|   ├── de-de.md  // 德文站首页   
|   └── index.md  // 默认首页
└── ...

Freezes 会知别到哪些文件是使用哪种语言的，当然这样做的话你也会容易在同一个站点管理多个语言版本的markdown 文稿内容。
