一切标准的 Markdown 文本可被 Freezes 用于生成站点，同样地 Freezes 内的 Markdown 可以使用任何作者喜爱的编辑器写作。

由于 Daring FireBarll 所制定的 Markdown 语法过于简结，并不能完全适应于生成丰富的网页内容。悻然，github 的流行并对 markdown 扩展语法 (Extra) 不遗余力的推广，markdown extra 已逐渐成为大众所认同的新的公共标准。因此 Freezes 除了支持 标准Markdown语法外还能支持以下的这些Markdown扩展格式：

代码扩展 ( fenced code blocks)

此扩展可生成带有样式类的代码块，与标准语法所不同的是没有4个空格或是1个tab的缩进占们符，取而代之的是以 ```样式类名+换行符(回车) 作为起始，以 ```+换行符(回车) 作为结束符

```class-name
 block code here
```

格式化代码输出后，Freezes 采用 highlight 对代码进行高亮格式化，可以支持24种常规语言格式，且可扩展至支持超过100种以上的开发语言。

目录索引

Freezes 不但简单地支持输出上下文索引目录，为了增加阅读的体验特别对目录索引的输出增强了滚动识别与自动高亮功能。

[TOC]

表格

标准markdown是没有表格格式的,表格扩展可支持以下的格式

First Header  | Second Header
------------- | -------------
Content Cell  | Content Cell
Content Cell  | Content Cell

通过对列声明对齐符

First Header | Second Header | Third Header
:----------- | :-----------: | -----------:
Left         | Center        | Right
Left         | Center        | Right

脚注

脚注经常用作引用注解，是优质文档不可缺失的内容。脚注可以随意标记，Freezes会统一将所有的脚注编排输出至页面的底部以适应人们普遍的阅读习惯。

这里需要注解的内容[^1]
[^1]:此处是对上述脚注标记的详细注释

在 [^1] 内 ^ 后面的内容是随意填写的，Freezes 会自动对脚注添加顺序号

删除线

格式：

~~~这是具有删除线的文字~~~

注 对于具有多个样式类的行内文字块请用第一种格式书写，以()括住多个样式名称，样式名称之间以空格隔开

锚点

锚点用于生成页内转跳的链接文本

## [这是一个例子](id:author1)

Freezes 可自动识别所有的合法的 URL 并自动生成链接标记
