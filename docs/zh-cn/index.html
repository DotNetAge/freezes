Freeze 是一个静态网站生成工具。如果您是一个专业作者，那么它就是一个专注于高质量写作体验，绝不以自定格式代码污染您的文档；如果您一位前端开发者，Freezes则是为你简化了一切常规化的配置工作，提供丰富的程序设计功能；

快速安装

只需要几步就能马上体验Freezes为您所带来的高质量写作体验：

终端

~ $ pip install freezes

~ $ mkdir my-web && cd $ _

~/my-web $ freezes new

~/my-web $ freezes

# => Now freezes will open browser to http://localhost:5500

为何使用 Freezes ?

我并不想重新发明轮子，奈何为了能使用 markdown 写作并共享到自己的站点时则遇到各种的问题：

  1. 大多博客工具都要安装和配置各种服务器环境，如数据库、配置 Web 服务器，仅仅共享文字信息则将大量时间花在配置上，同时又要为特定的程序运行环境和数据库空间付费。
  2. 基于Web的 markdown 工具与本地化的 markdown 工具的使用体验实在相差太远。
  3. 为了让站点变得更加美观吸引要么自己动手学习博客平台的各种定制化要求，要么就得到处寻找特定的主题样式。
  4. 并不是所有的主题外观都能支持提供良好的阅读体验，由其是文字的格式与代码的格式。
  5. 不知是受到过时的WIKI影响还是为了开发方便，很多支持markdown或基于markdown写作的静态化站点工具要我在 markdown 文本内加入他们定制的特定标记，这无疑是一种污染，一种极大增加额外工作量的用法。

为此我开发了 Freezes , 一个可以用纯净的markdown写作，没有自定标记混杂，允许使用各种的 markdown 编辑器，可以随意使用 less , sass , coffee , javascript 自已动手设计与美化站点的静态化站点工具！

Freezes 的 设计目标 ：

  • 对于作者可以用自己喜爱的 markdown 工具,将时间投注于写作之中。
  • 对于网页设计者远离繁杂的配置、远离陡峭的学习曲线、将已有的设计技术淋漓尽致地发挥如一。
  • 无配置部署，在最廉价的服务器上共享美观的站点与高质量内容。
