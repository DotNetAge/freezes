站点各种目录与文件的详细说明

在运行 freezes new 指令后，Freezes 将自动在当前目录下创建站点所需要运行的各种目录和文件，用 Finder 或 文件管理器 打开的话其目录结构如下：

.
├── _config.yml
├── _assets.yml
├── pages
|   ├── posts
|   |     └── 2014-01-01
|   |           ├─ why-programmer-should-write-books.md
|   |           └─ why-programmer-should-write-books
|   ├── documents     
|   |   
|   └── index.md
├── builds
├── static
|   ├── images
|   ├── scripts
|   └── styles
├── templates
|   ├── includes
|   ├── layouts
|   └── 404.html
└── __init__.py

以下是对所有的默认目录的具体说明

目录/文件 作用 说明
pages 工作目录 存放所有的写作的文件，你可以建立各种你需要的目录对文件进行分类(详细请参考"内容组织")
pages/posts 随笔(博客)目录
pages/index.md 首页 不可删除
static 静态资源目录
static/images 图片资源目录
static/scripts 脚本目录
static/styles 样式目录
templates 页面模板目录
templates/includes 存放包含文件
templates/layouts 存放布局页面文件
builds 生成目录 站点静态化后的最终文件输出目录
_config.yml 站点配置文件 详细配置请参考："配置说明"
_assets.yml 资源配置
