环境要求

在开始安装Freezes之前，有一些基本的环境要求

  • Python 2.7
  • pip
  • virtualenv
  • Linux, Unix, or Mac OS X
  • CoffeeScript
  • Less
  • Sass (可选，用于支持 Sass资源).

Python

Freezes 是基于 Python 2.7 环境下开发，OSX已搭载 python 2.7 而无需安装，对于 Linux (Ubuntu) 则需要运行以下指令

sudo apt-get install python-dev

pip 和 easy_install

Freezes 的安装需要运行 pip 和 easy_install 指令，它们工具并不是python中的标准搭载工具，OSX和Linux用户可以分别按以下方式安装

Mac OSX 和 Windows 用户需要在先下载 get-pip.py 文件，然后运行

python get-pip.py

Debian 和 Ubuntu 

sudo apt-get install python-pip

Fedora 

sudo yum install python-pip

virtualenv

Python的背后有着庞大的开源社区支持，但是有一个缺点就是每个包的质量都参差不齐，如果我们在工作服务器上去测试安装每个包，就会造成整个的服务器形成庞大复杂的第三方包依赖。

virtualenv 就是为了解决这个问题而生，他可以在我们的目录上生成若干个独立的虚拟环境，这些虚拟环境和正式环境一样。因为了避免与您机器上其它正在使用python环境运行的项目或程序产生冲突，我们强烈地建议你使用 virtualenv 建立一个隔离、清结的python环境。

如果你正在使用 Mac OSX 那么只需要在终端内运行以下的指令

$ sudo easy_install virtualenv

或者

$ sudo pip install virtualenv

如果你正在使用 Linux (Ubuntu) 那么请尝试：

$ sudo apt-get install python-virtualenv

当 virtualenv 安装成功后，在使用Freezes建立您的站点之前请用以下指令激活 virtualenv 环境:

$ mkdir <你的站点目录名> && cd $_
<你的站点目录名> $ virtualenv venv
<你的站点目录名> $ source venv/bin/activate

成功激活后会在命令符之前出现 (venv)：

(venv)<你的站点目录名> $

此时就可以使用 pip 工具安装 Freezes 了

Less

默认情况下 Freezes 是使用 Less 开发网站样式的，Freezes的自带默认样式也是采用 Less 开发的。如果您的机器上没有安装 Less 的编译器可能会导致 Freezes 无法正确为您将 *.less 样式资源生成最终的 css样式表。

为了避免这个情况发生，那么请使用 npm 安装一个less的编译器到您全局环境中:

Mac OSX

$ npm install -g less

Linux Ubuntu

$ sudo apt-get install node-less

如果你想用想用less那么请在资源配置文件( 网站目录/_assets.yml) 内将 css 的配置按以下的方式修改:

css:
    output: "your-website-root/site.min.css"
    contents: "your-website-root/styles/*.css"
    filters: cssprefixer,cssmin

your-website-root 是当前目录的名称，Freezes 建立站点时已为您设置好，切勿直接将代码直接拷贝。

CoffeeScript

采用CoffeeScript 写脚本可以比直接写javascript节省2/3以上的代码量，所以Freezes是默认支持 CoffeeScript 的。同上，如果您还没有安装CoffeeScript编译器，那么请运行以下的指令：

Mac OSX

$ npm install -g coffee-script

Linux Ubuntu

$ sudo apt-get install coffeescript

如果你想采用javascript作脚本开发可以在资源配置文件( 网站目录/_assets.yml) 内将 js 的配置按以下的方式修改

js:
    output: "your-website-root/site.min.js"
    contents: "your-website-root/scripts/*.js"
    filters: jsmin
