How to reference link to pages and resources

  • Home

  • 订阅

传统的WIKI或者博客为了编程方便往往喜爱将文本所引用到的图片或者是视频资源统一放置在一个目录内，只要引用的资源越来越多，文章也越来越多之间几乎是无法找到与文章引用对应的资源文件的，这无疑为文稿相关资源的管理带来困难。

Freezes允许作者使用最常规的文件与资源管理手段就是通过目录进行分类与资源管理。当文章需要引用额外的图片或者是其它资源时，并无需指明图片的具体地址，而只需要声明其文件名，同时在文章所在的目录中建立一个同名的文件夹，将资源文件放置其中即可引用成功。

例如，需要在quick-start.md文件内引用一个cover.png图片,目录结构则应该如下：

.
├── pages
|   ├── posts
|   |     └── 2014-01-01
|   |           ├─ quick-start
|   |           |      └─ cover.png
|   |           └─ quick-start.md
|   |   
|   └── ...
└── ...

那么在 quick-start.md 文件内就可以以下方式引用 cover.png 文件:

![Cover](cover.png)

同理，Freezes 对一切非 *.md 后缀文件放置于 pages 目录都可以上文的方式引用。

链接其它的文稿

我们在写作时经常会链接到其它的相关文稿上，Freezes 是采用相对目录的方式引用至其它文稿。我们平时就是在目录中管理文件，所以通过相对目录引用相关文件是最自然的做法。

例如：在 “我的专栏” (columns.md) 文件内添加一链接指向 my-design-philosopy.md

.
├── pages
|   ├── columns // 我的专栏下的文稿
|   |     ├── my-design-philosophy.md
|   |     ├── ...
|   |     └── why-programmer-should-write-books.md
|   ├── columns.md  // 我的专栏   
|   └── index.md
└── ...

在columns.md文稿的正文内则应该像如下方式处理：

请参考：[我的设计哲学](my-design-philosophy)

反之，如果在 my-design-philosophy.md 文稿内引用 columns.md 的话则应该采用 / 以加入全路径:

请参考：[我的专栏](/columns/)
